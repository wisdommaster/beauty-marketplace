## Протокол взаимодействия

```
Оператор → Оркестратор: задача на русском
Оркестратор → Агент: sessions_spawn(task, model, context="isolated")
Агент: выполняет, коммитит в GitHub
Агент → Оркестратор: результат + отчёт
Оркестратор → Оператор: консолидация + запрос ревью
```

### Правила

- Агенты НЕ общаются напрямую
- Вся координация — через Оркестратора
- Каждый агент: изолированный контекст, свой набор MCP/API
- Результат: код в репо + краткий отчёт в Lethe
- Ошибка: флаг оператору, не маскировать

### Очерёдность запуска

```
1. Creative & Brand  ← СЕЙЧАС (все API готовы)
2. Infrastructure    ← после получения VPS
3. Backend Builder   ← после получения YClients/YooKassa/Яндекс ключей
4. Frontend Builder  ← после готовности /api/feed + /api/bookings
5. Marketing         ← за 1 неделю до пилота (Phase 3)
```
