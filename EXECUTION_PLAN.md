## Оркестрация агентов

### Архитектура

```
Оператор (человек)
    │
    ├── Оркестратор (Agatha — main agent)
    │       │
    │       ├── Backend Agent     (Node.js/TypeScript, PostgreSQL, YClients API, YooKassa)
    │       ├── Frontend Agent    (React, WebSocket, адаптивная вёрстка)
    │       ├── DevOps Agent      (VPS, Docker, nginx, CI/CD, домен, SSL)
    │       └── Marketing Agent   (VK Ads, Яндекс.Директ, метрики)
    │
    └── Спецификация → передаётся агенту на исполнение
```

### Backend Agent

**Стек:**
- Node.js + TypeScript
- PostgreSQL (основная БД)
- Redis (сессии, кэш, WebSocket pub/sub)
- Docker
- YClients REST API v2
- YooKassa API
- Yandex Maps API (Geocoder, Places)

**MCP-серверы / API-интеграции:**
- `yclients-mcp` — прослойка над YClients API
- `yookassa-mcp` — платёжные операции
- `yandex-maps-mcp` — геокодинг, рейтинги, маршруты

**Инструкция:** следовать спецификации из `projects/beauty-marketplace/`. Код — чистый, типизированный, с тестами. Каждый PR — ревью оркестратором.

### Frontend Agent

**Стек:**
- React + TypeScript
- Tailwind CSS
- WebSocket-клиент
- PWA (офлайн-поддержка)

**Инструкция:** мобильно-первый дизайн. Скелетоны при загрузке. Все состояния (ошибка, пусто, загрузка, край случая) — реализованы до красоты.

### DevOps Agent

**Стек:**
- VPS (Ubuntu 24.04, 4 vCPU, 8 GB RAM)
- Docker Compose
- nginx reverse proxy
- Let's Encrypt SSL
- GitHub Actions CI/CD
- Мониторинг: Prometheus + Grafana

**Инструкция:** авто-деплой из main-ветки. Бекапы БД раз в сутки. Логи в централизованный сборщик.

### Marketing Agent

**Каналы:**
- Яндекс.Директ (поиск: «массаж со скидкой СПб», «спа сегодня СПб»)
- VK Ads (таргет: женщины 25-45, СПб, интересы beauty/wellness)
- Instagram / Telegram-каналы района

**Метрики:**
- CAC (стоимость привлечения пользователя)
- Конверсия установка → регистрация → первая бронь
- Удержание D7, D30

## Риски и митигации

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| YClients API сломает интеграцию | Средняя | Высокое | Кэш слотов на 1ч, graceful degradation |
| Три студии — слишком узкая лента | Высокая | Высокое | Гео-запуск в одном районе (Петроградский), интенсивный набор студий в первую неделю пилота |
| YooKassa откажет в подключении | Низкая | Среднее | Старт с OFFLINE_ONLY, YooKassa — фаза 2 |
| Пользователи не понимают модель скидки | Средняя | Среднее | Раскрытие формулы на карточке, FAQ, онбординг |
| Конкуренты скопируют | Средняя | Низкое | Быстрый захват рынка СПб, бренд, сетевой эффект |
