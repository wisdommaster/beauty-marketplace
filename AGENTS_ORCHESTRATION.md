## Взаимодействие агентов

```
Оператор (человек)
    │
    ├── Сессия с Оркестратором (Agatha)
    │       │
    │       ├─ Задача → Backend Agent
    │       │     Статус: ждёт YClients-ключи
    │       │
    │       ├─ Задача → Frontend Agent
    │       │     Статус: ждёт готового API
    │       │
    │       ├─ Задача → DevOps Agent
    │       │     Статус: готов к запуску (VPS)
    │       │
    │       ├─ Задача → Marketing Agent
    │       │     Статус: готов к контент-плану
    │       │
    │       └─ Задача → Creative Agent
    │             Статус: готов (брендинг)
    │
    └── Сессия с агентом (через sessions_spawn)
          │
          ├─ Изолированный контекст
          ├─ Свой набор инструментов (скиллы + MCP)
          └─ Результат → обратно Оркестратору
```

### Протокол взаимодействия

1. **Оператор → Оркестратор:** задача на естественном языке
2. **Оркестратор → Агент:** `sessions_spawn(task, model, skills)` с конкретным ТЗ
3. **Агент:** выполняет, коммитит в репо, докладывает
4. **Оркестратор:** проверяет результат, передаёт следующую задачу или оператору на ревью

### Правила агентов

- Один агент = одна зона ответственности
- Не пересекаются по файлам (Backend ≠ Frontend)
- Коммуникация только через Оркестратора (не напрямую)
- Результат: код в репо + краткий отчёт
- При ошибке: флаг оператору, не замалчивать
