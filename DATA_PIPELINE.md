# От YClients до карточки в ленте — Полный пайплайн

---

## 1. Что приходит из YClients

### Сырые данные (API YClients v2)

**Услуга (`get_services`):**
```
service_id: 12345
title: "Хатха-йога"
price: 2400                    // базовая цена, целое число
duration: 5400                 // секунд (90 мин)
category: "Йога"               // категория в YClients — не стандартизирована
staff_id: 678
```

**Сотрудник (`get_staff`):**
```
staff_id: 678
name: "Анна Смирнова"
specialization: "Хатха, Виньяса, Йога-терапия"
position: "Инструктор"         // может быть пустым или произвольным
```

**Слот (`get_available_days` + `get_available_times`):**
```
date: "2026-04-28"
time: "18:00"
datetime: "2026-04-28T18:00:00+03:00"
staff_id: 678
service_id: 12345
is_free: true
capacity: 1                    // 1 = индивидуальное, >1 = групповое
booked_count: 0
```

**Студия (из профиля YClients):**
```
company_id: 999
name: "Yoga Space Studio"
address: "ул. Арбат, 12, Москва"
phone: "+7..."
```

## 2. Чего НЕТ в YClients — нужно обогатить

| Данные | Есть в YClients? | Откуда берём |
|---|---|---|
| Категория платформы (Вэлнес/Бьюти/Спа/Спорт) | ❌ Нет | Бизнес выбирает при настройке услуги |
| Формат (индивидуальное/групповое) | ⚠️ Частично (capacity) | Авто-mapping из capacity: 1→индивидуальное, >1→групповое |
| Уровень (начинающий/средний/продвинутый) | ❌ Нет | Бизнес настраивает вручную |
| Что взять с собой | ❌ Нет | Бизнес заполняет вручную |
| Описание услуги (развёрнутое) | ❌ Нет | Бизнес заполняет вручную |
| Описание студии | ❌ Нет | Бизнес заполняет при регистрации |
| Фото студии | ❌ Нет | Бизнес загружает вручную |
| Рейтинг | ❌ Нет | Считает платформа по отзывам |
| Роль сотрудника | ⚠️ Ненадёжно | Авто-mapping по категории |
| Опыт сотрудника (лет) | ❌ Нет | Бизнес заполняет вручную |

## 3. Пайплайн: от слота YClients до карточки в ленте

### Шаг 1: Синхронизация (поллинг YClients)

```
Каждые 60 секунд (incremental) или 15 минут (full sync):

1. Платформа → YClients API:
     GET /company/{id}/services
     GET /company/{id}/staff
     GET /company/{id}/available_days
     Для каждого дня: GET /company/{id}/available_times

2. YClients → Платформа: сырые данные

3. Платформа: UPSERT в свою БД
```

### Шаг 2: Автоматическое обогащение (Auto-mapping)

На стороне платформы, **сразу при получении данных**:

| Правило | Вход (YClients) | Выход (Платформа) |
|---|---|---|
| Если `capacity == 1` | capacity=1 | format = "individual" |
| Если `capacity > 1` | capacity=6 | format = "group" |
| Категория услуги уже настроена бизнесом → использовать | business_mapping: service_12345 → "Вэлнес" | category = "wellness" |
| Категория НЕ настроена → пусто, услуга не публикуется | — | category = NULL → hidden |
| `price` в копейках/рублях → нормализация | price=2400 | base_price = 2400₽ |
| `duration` в секундах → минуты | duration=5400 | duration_min = 90 |

### Шаг 3: Ручное обогащение бизнесом (один раз при подключении)

Бизнес заходит в `/business/services` после синхронизации и видит:

```
Услуги из YClients (4 загружено, 2 настроены):

✅ Хатха-йога        2 400₽  90 мин  Инструктор: Анна Смирнова
   Категория: [Вэлнес ▾]  Формат: инд./групп.  Уровень: [Любой ▾]
   Описание: [Хатха-йога с акцентом на дыхание...]
   Что взять: [Коврик, вода]
   [Сохранить]

⚠️ Виньяса-йога      2 800₽  75 мин  Инструктор: Анна Смирнова
   ! Не настроена — не показывается в ленте
   [Настроить]

❌ Массаж лица       3 500₽  60 мин  Мастер: Елена Петрова
   ! Не настроена
   [Настроить]
```

**Что бизнес заполняет вручную (один раз):**

| Поле | Тип | Обязательно |
|---|---|---|
| Категория платформы | Выпадающий (4 варианта) | ✅ Да |
| Формат | Авто из capacity, можно переопределить | ✅ Да |
| Уровень | Выпадающий (Любой/Начинающий/Средний/Продвинутый) | Опционально |
| Описание услуги | Текстовое поле | Опционально |
| Что взять с собой | Текстовое поле | Опционально |
| Роль сотрудника | Авто из категории, можно переопределить | ✅ Да |
| Опыт сотрудника | Число + «лет» | Опционально |

### Шаг 5: Попадание в ленту пользователя

Пользователь открывает `/app/feed`. Платформа:

```
1. Фиксирует геолокацию пользователя: lat, lon

2. Вычисляет расстояние до каждого доступного слота:
     distance = haversine(user_lat, user_lon, slot_lat, slot_lon)

3. Применяет фильтры:
     ✅ slot.is_available == true
     ✅ slot.booked_count < slot.capacity
     ✅ start_time внутри временного окна (+12ч для Growth, +24ч для Maximum)
     ✅ distance ≤ min(slot.max_radius, user_radius)
     ✅ slot.category входит в выбранные пользователем (из предпочтений)
     ✅ slot.start_time - now > 0 (не в прошлом)

4. Вычисляет скидку:
     discount = slot.min_discount + (distance / slot.max_radius) × (slot.max_discount - slot.min_discount)
     final_price = slot.base_price × (1 - discount / 100)

5. Строит карточку для ленты:
     → category_pin: "🔵 Вэлнес"
     → service_name: "Хатха-йога"
     → start_time: "18:00–19:30"
     → base_price: "~~2 400₽~~"  
     → final_price: "1 680₽"
     → discount_badge: "−30%"
     → distance: "3.2 км"
     → address: "ул. Арбат, 12"
     → rating: "⭐ 4.8"
     → studio_name: "Yoga Space Studio"

6. Композиция пачки (алгоритм из PLATFORM_SPEC.md):
     → Preferred слоты (40%)
     → Смежные категории (30%)
     → Discovery (20%)
     → Wildcard (10%)

7. Отдаёт пачку из 10-12 карточек пользователю
```

### Шаг 6: Обновление статусов внутри 30-мин окна

```
Карточка показана. Поллинг (60с) проверяет:

1. YClients: изменился ли booked_count?
     → Если да → обновить booked_count на платформе
     → Если booked_count >= capacity → slot.is_available = false
     → WebSocket / SSE: push обновления всем пользователям с этой карточкой

2. Платформа: не истекло ли время слота?
     → Если start_time < now → slot.is_available = false

3. Платформа: не истекло ли временное окно?
     → Если start_time - now > окно_тарифа → скрыть карточку
```

## 5. Валидация при синхронизации

Что платформа проверяет при каждом пайплайне:

| Проверка | Если не проходит |
|---|---|
| Услуга настроена бизнесом (категория не NULL) | Не публикуется в ленте |
| `base_price > 0` | Не публикуется |
| `start_time > now` | Не публикуется |
| `start_time` в пределах временного окна тарифа | Не публикуется (будет показана позже) |
| `booked_count < capacity` | Показывается как доступная |
| `booked_count == capacity - 1` | Бейдж «1 место» |
| `booked_count == capacity` | Серая, недоступна |
| Студия имеет активный тариф | Публикуется |
| Тариф истёк / не оплачен | Скрывается из ленты |
| Студия не заблокирована модерацией | Публикуется |

## 8. Спецификация auto-mapping полей

### Формат (capacity → format)

```python
def map_format(capacity: int) -> str:
    if capacity == 1:
        return "individual"
    return "group"
```

Бизнес может переопределить. Например: массаж — всегда индивидуальный, даже если в YClients стоит capacity=2.

### Роль сотрудника (категория → роль)

```python
ROLE_MAPPING = {
    "wellness": "Инструктор",
    "beauty": "Мастер",
    "spa": "Специалист",
    "sport": "Тренер"
}
```

Бизнес может переопределить. Например: «У нас мастера йоги, не инструкторы».

### Длительность (секунды YClients → минуты)

```python
def map_duration(seconds: int) -> int:
    return seconds // 60
```

Не переопределяется.

### Время окончания

```python
end_time = start_time + timedelta(minutes=duration_min)
```

Не переопределяется.

## 10. Контрольные точки валидации

| Точка | Что проверяем | Действие при ошибке |
|---|---|---|
| YClients API недоступен | Коннекшн / 5xx | Лог, retry через 60с, стейл-данные из кэша |
| Новый слот | Все поля не NULL | Пропускаем, логируем |
| Категория не назначена | category == NULL | Слот скрыт, бизнесу уведомление |
| Цена = 0 | base_price <= 0 | Слот скрыт, бизнесу: «Проверьте цену в YClients» |
| Дубликат слота | Тот же yclients_slot_id уже в базе | UPSERT, не дублируем |
| Статус изменился (поллинг) | booked_count изменился | Обновляем, пушим WebSocket |
