# Beauty/Wellness Marketplace — Полная архитектура

## 1. Концепт

Маркетплейс для beauty/wellness-бизнесов (салоны, спа, йога, массаж, косметология). Помогает заполнять пустые слоты в расписании через радиусно-зависимую динамическую скидку.

**Ключевая идея:** Чем дальше пользователь от студии — тем больше скидка на пустой слот. Скидка = награда за готовность ехать дальше.

**Для бизнеса:** дополнительная выручка с пустых слотов без затрат на маркетинг.

**Для пользователя:** скидка на услуги сегодня/завтра, линейно растущая с расстоянием.

---

## 2. Формула скидки

```
discount = бизнес_min_discount + (distance / бизнес_radius) × (бизнес_max_discount - бизнес_min_discount)
final_price = base_price × (1 - discount)
```

- **Min/max discount — всегда от бизнес-тарифа.** Пользовательский радиус — ТОЛЬКО фильтр видимости. На формулу не влияет.
- При любом пересечении тарифов скидка считается одинаково для всех пользователей на одном расстоянии.
- Premium покупает охват (видит дальше), а не лучшую скидку.
- Отсчёт расстояния — от студии (бизнес — центр)
- Haversine для прямой дистанции (MVP). Далее — routing API (Google/2GIS/OSRM) для дорожного расстояния

**Пример:** Бизнес Maximum (25 км, 15-50%). Расстояние 4 км. Базовая цена 5000₽.
- discount = 15% + (4/25) × 35% = 15% + 5.6% = 20.6%
- Цена = 5000 × (1 - 0.206) = 3 970₽
- Одинаково для Free, Comfort, Premium — при условии что все видят это предложение (4 км ≤ их радиус)

Бизнес задаёт радиус показа. Пользовательский радиус — ТОЛЬКО фильтр. Скидка — всегда от бизнес-тарифа, по бизнес-радиусу.

| Бизнес \\ Пользователь | Free (7 км) | Comfort (20 км) | Premium (∞) |
|---|---|---|---|
| **Start** (3 км, 5-20%) | 0-3 км ✅ | 0-3 км ✅ | 0-3 км ✅ |

## 4. Монетизация платформы

**Только тарифы.** Ноль комиссии с транзакций.

- Бизнес: 0₽ / 3 990₽ / 7 990₽ в месяц
- Пользователь: 0₽ / 299₽ / 599₽ в месяц
- Доп. фичи: приоритет в ленте, аналитика, мульти-локация, API-доступ
- Контракты: 3/6/12 мес со скидкой 20/30/40%

## 5. Интеграция с YClients

**Маркетплейс — source of truth для пользователя. YClients — CRM салона.**

- Салон работает только в YClients (как привык)
- Платформа: своя БД слотов, синхронизированная с YClients (full sync раз в 15 мин, incremental раз в 60с / webhook)
- Пользователь не касается YClients
- Салон может подтвердить визит в YClients ИЛИ в админ-панели платформы
- Маркетплейс получает статусы от YClients через поллинг (30с)

```
Пользователь → Маркетплейс (бронь + оплата)
                    │
                    ├─ YooKassa (платёж, 100% бизнесу)
                    │
                    └─ YClients (только синхронизация)
                              ↑
                           Салон
```

## 6. Бронирование и статусная модель

### Статусы брони

```
HELD ──┬──▶ CANCELLED       (таймаут / ручная отмена >30 мин до начала)
       ├──▶ LATE_CANCEL      (отмена ≤30 мин до начала)
       ├──▶ PAID             (ONLINE_REQUIRED: платёж прошёл)
       │     ├──▶ CANCELLED  (отмена >30 мин, 100% refund)
       │     ├──▶ LATE_CANCEL (отмена ≤30 мин, 0% refund)
       │     └──▶ CONFIRMED → COMPLETED
       └──▶ CONFIRMED        (OFFLINE_ONLY: салон подтвердил)
             └──▶ COMPLETED
```

### Таймауты

- ONLINE_REQUIRED: 5 минут на оплату → CANCELLED
- OFFLINE_ONLY: 10 минут на подтверждение намерения → CANCELLED

### Отмена

- >30 мин до начала: 100% возврат (ONLINE), без штрафа (OFFLINE)
- ≤30 мин до начала: 0% возврат (ONLINE), LATE_CANCEL (OFFLINE)
- No-show: 3 штрафа за 30 дней = ограничение (бан / только предоплата)

---

## 7. Режимы оплаты

### OFFLINE_ONLY
- Бронь без оплаты → HELD → салон подтверждает → CONFIRMED
- Оплата на месте, платформа не участвует в деньгах

### ONLINE_REQUIRED
- 100% предоплата через YooKassa
- YooKassa → прямой перевод бизнесу (100%, без комиссии платформы)
- Платформа получает webhook о статусе платежа
- Статус PAID → мгновенно на маркетплейсе, YClients — асинхронно

## 8. Лента и ротация карточек

### Механика
- Пользователь видит пачку из 10-12 карточек
- Геолокация фиксируется на 30 минут
- Seed = user_id + date + batch_number (гарантирует разный набор при ротации)
- Batch меняется каждые 30 мин (авто) или при pull-to-refresh

### Композиция пачки
| Слоты | Тип | Приоритет |
|---|---|---|
| 1-4 | Preferred категории | ×1.5 |
| 5-7 | Смежные категории | ×1.0 |
| 8-9 | Discovery | ×0.8 |
| 10 | Wildcard (макс. скидка) | — |

Пост-проверки: ≤2 карточки от одной студии, ≤60% одной категории, ≥2 разных района.

### Статусы слотов внутри 30-мин окна
- Карточки не обновляются (фиксированный набор), но статусы — живые
- Поллинг YClients каждые 60с по всем slot_id в пачке
- Если слот занят → карточка сереет, «только что забрали»

### Carry-over при ротации
- 1-2 карточки из предыдущей пачки (если слот ещё доступен)
- Приоритет в новой пачке выше
- Создаёт непрерывность опыта

---

## 9. Атомарность и защита от race condition

- БД платформы: SELECT FOR UPDATE при захвате слота
- YClients create_booking() — idempotency key для защиты от двойного бронирования при ретраях
- Групповые слоты: проверка booked_count < capacity атомарно в транзакции
- При конфликте (слот заняли параллельно): показ альтернатив, не ошибка

---

## 10. Ключевые риски и митигации

| Риск | Митигация |
|---|---|
| Каннибализация полной цены | Временное окно +12ч, скидки только на ближайшие слоты |
| Холодный старт (мало предложений) | Плотный гео-запуск: 20-30 студий в одном районе |
| Погрешность haversine vs дорожное расстояние | Routing API (Google/2GIS/OSRM) с первой версии |
| Спуфинг GPS | Множественная проверка (GPS + IP + WiFi), лимиты |
| YClients API недоступен | Кэш слотов, catch-up при восстановлении |

## 11. План запуска (MVP)

### Фаза 0: Пилот (2-4 недели)
- 5-10 студий в одном районе Москвы
- Ручная координация через Telegram/WhatsApp
- Сбор данных: конверсия, средний чек, частота использования

### Фаза 1: MVP (2-3 месяца)
- Интеграция YClients API
- Мобильное приложение (iOS + Android)
- Базовые тарифы
- Офлайн-оплата (OFFLINE_ONLY)
- Один город, 3 района

### Фаза 2: Рост
- Онлайн-оплата (YooKassa)
- Пользовательские тарифы
- Расширенная аналитика
- Второй город
