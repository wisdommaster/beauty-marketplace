## Стратегия токенизации

### Правила экономии токенов

1. **Промпты — один раз, контекст — минимальный**
   - Не передавать всю спецификацию агенту. Только конкретную задачу + ссылку на документ.
   - Использовать `lightContext: true` для повторяющихся задач.

2. **Кэширование системных промптов**
   - DeepSeek V4 Pro: cache hit = $0.0028/M (в 100× дешевле). Кэшировать спецификацию как системный промпт.
   - Claude: prompt caching доступен для длинных контекстов.

3. **Иерархия моделей**
   ```
   Задача простая (скрипты, миграции, конфиги)
     → DeepSeek V4 Flash ($0.28/M out)
   
   Задача средняя (CRUD, роуты, компоненты)
     → DeepSeek V4 Flash OR GPT-5 если нужна надёжность
   
   Задача сложная (архитектура, ревью, интеграции)
     → Claude Sonnet ($15/M out) — только когда Flash не справляется
   ```

4. **Батчинг задач**
   - Вместо 10 отдельных запросов к Claude → 1 батч-запрос с 10 задачами
   - Экономия на повторяющихся системных промптах: 50%

5. **OpenAI DALL-E: эконом-режим**
   - Генерация через Together AI для драфтов ($0.01/изобр.)
   - DALL-E 3 только для финальных вариантов ($0.04-0.12/изобр.)
